# 餐具检测系统 (Tableware Detection System)

## 项目概述

这是一个基于OpenCV的餐具质量检测系统，采用计算机视觉技术自动判断餐具的质量状态（OK/NG）。系统通过HSV颜色空间分析、形态学处理、连通域分析等图像处理技术，实现对餐具缺陷的自动检测。

## 核心功能

- **自动质量检测**: 对餐具图像进行自动分析，输出OK/NG结果
- **图像预处理**: 智能裁剪图像，保留关键检测区域
- **HSV颜色分析**: 基于色相和饱和度进行缺陷区域分割
- **形态学处理**: 去噪声和填充处理，提高检测精度
- **连通域过滤**: 基于面积百分比过滤小干扰区域
- **实时结果显示**: 多窗口显示处理过程和最终结果
- **交互式颜色分析**: 鼠标点击获取像素颜色信息
- **批量处理**: 支持单张和批量图像处理

## 技术架构

### 核心模块

#### 1. 主程序 (`main.cpp`)
- 命令行参数解析
- 图像读取和预处理
- 处理流水线控制
- 结果保存和显示
- 性能计时统计

#### 2. 图像处理模块 (`image_processing.cpp/h`)
核心算法函数：
- `createHueBinaryMask()`: HSV二值化分割
- `performMorphological()`: 形态学处理
- `fillContours()`: 轮廓填充
- `filterConnectedComponentsByPercent()`: 连通域过滤
- `findAndDrawMaxColumn()`: 缺陷列检测和OK/NG判定

#### 3. 显示模块 (`display.cpp/h`)
用户界面功能：
- `createSubplotDisplay()`: 多图像子窗口布局
- `showColorAnalysis()`: 交互式颜色分析窗口
- `onMouse()`: 鼠标事件处理

#### 4. 配置模块 (`config_constants.h`)
可调参数配置：
- HSV颜色检测阈值
- 形态学处理参数
- 连通域过滤参数
- 质量判定阈值
- 图像裁剪比例

## 检测算法流程

```
原始图像 → HSV转换 → 二值化分割 → 形态学处理 → 连通域过滤 → 列分析 → OK/NG判定
```

### 详细步骤

1. **HSV转换**: 将BGR图像转换为HSV颜色空间
2. **二值化分割**: 基于H通道(8-20)和S通道(≥150)进行颜色分割
3. **形态学处理**: 
   - 开运算(5×5核): 去除噪声
   - 闭运算(19×19核): 填充空洞
4. **连通域过滤**: 过滤面积小于全图2%的小区域
5. **缺陷分析**: 找到白色像素最多的列，基于位置和比例判定质量

## 编译和运行

### 系统要求

- **操作系统**: Windows 10/11
- **编译器**: Visual Studio 2019 或更新版本
- **CMake**: 版本 3.10 或更高
- **OpenCV**: 4.8.1 (项目内已包含)

### 编译步骤

1. **自动编译**（推荐）
   ```bat
   build.bat
   ```

2. **手动编译**
   ```bat
   mkdir build
   cd build
   cmake .. -G "Visual Studio 16 2019" -A x64
   cmake --build . --config Release
   ```

### 运行程序

#### 单张图像检测
```bat
build\Release\tableware_detection.exe image_path.jpg
```

#### 批量测试
```bat
test.bat
```

测试脚本提供两种模式：
- **模式1**: 单张图片测试
- **模式2**: 文件夹批量处理

## 配置参数

主要参数在 `config_constants.h` 中配置：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `HSV_H_MIN` | 8 | HSV色相最小值 |
| `HSV_H_MAX` | 20 | HSV色相最大值 |
| `HSV_S_MIN` | 150 | HSV饱和度最小值 |
| `MORPH_OPEN_KERNEL_SIZE` | 5 | 开运算核大小 |
| `MORPH_CLOSE_KERNEL_SIZE` | 19 | 闭运算核大小 |
| `CONNECTED_COMPONENT_PERCENT` | 2.0 | 连通域面积阈值(%) |
| `PIXEL_RATIO_THRESHOLD` | 0.4 | 像素比例阈值(40%) |
| `POSITION_OFFSET_MIN_THRESHOLD` | 0.0 | 最小位置偏移阈值(0%) |
| `POSITION_OFFSET_MAX_THRESHOLD` | 0.15 | 最大位置偏移阈值(15%) |


## 输出结果

### 显示窗口
程序会显示包含6张子图的处理结果窗口：
1. 原始图像
2. HSV图像
3. 二值化结果
4. 形态学处理结果
5. 连通域过滤结果
6. 列检测结果

### 交互式颜色分析
处理完成后，程序会显示交互式颜色分析窗口：
- **鼠标移动**: 实时显示鼠标位置的颜色信息
- **像素信息**: 显示坐标、HSV值、RGB值和灰度值
- **可视化标记**: 绿色十字线标记当前位置
- **ESC键**: 退出程序

### 保存文件
自动保存以下结果文件：
- `morphological_processing_*.jpg`: 完整处理结果
- `binary_mask_*.jpg`: 二值化结果
- `morphological_*.jpg`: 形态学处理结果
- `final_result_*.jpg`: 最终过滤结果

### 质量判定
- **OK**: 绿色显示，表示餐具质量合格
  - 像素密度 ≥ 40%
  - 位置偏移在 0% ~ 15% 范围内
- **NG**: 红色显示，表示检测到缺陷
  - 像素密度不足或位置偏移超出范围

## 故障排除

### 常见问题

1. **编译失败**
   - 检查Visual Studio 2019是否正确安装
   - 确认CMake版本≥3.10

2. **运行时DLL缺失**
   - 编译会自动复制所需DLL文件到输出目录
   - 确保在build\Release目录下运行程序

3. **图像无法加载**
   - 检查图像文件路径是否正确
   - 支持格式：jpg, jpeg, png, bmp, tiff, tif

4. **检测结果不准确**
   - 调整`config_constants.h`中的HSV阈值参数
   - 根据具体应用场景优化判定阈值
